Priority Level: Medium
Recommended Customer Count: 3-5 customers
Development Stage: Phase 2
Estimated Build Time: 1 week (with global amenities)
Purpose & Use Case:

Let individual accounts customize their amenities list from the global master list (admin-settings-amenities.html). Each account can: enable/disable specific amenities, override adjustment values for their local market, add custom account-specific amenities. Example: Miami account might value "Pool" at $15K while Seattle account values it at $8K.

Key Features Explained:
  • Enable/Disable Toggle: Accounts only see amenities they've enabled. Keeps their interface clean.
  • Custom Adjustment Values: Override global default for local market. Example: "Fireplace" worth more in cold climates.
  • Inherited from Master List: Grayed amenities use global default values. Custom values shown in green/bold.
  • Search & Filter: Find specific amenities in long lists. Filter by category or status.
  • Add Custom Amenity: Account-specific amenities not in global list. Only this account sees them.
  • Statistics: Shows enabled count vs total available. Helps see customization level.
Implementation Notes:

Built WITH admin-settings-amenities.html as a pair. Database: account_amenities (account_id, amenity_id, is_enabled, custom_adjustment_value, is_custom). Query shows all global amenities + account's custom ones. Toggle updates is_enabled. Adjustment value updates only affect this account. Custom amenities have NULL amenity_id (or separate table). Most accounts will just use defaults - customization is optional power feature.
