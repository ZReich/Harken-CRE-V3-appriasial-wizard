BUILD NOTES - Bulk Upload Duplicate Review

Priority Level: High
Development Stage: Prototype
Purpose:

When users bulk upload comps, this screen appears if any uploaded addresses match existing records. Users can review each potential duplicate and decide: create as new record, update existing record, or skip. This prevents silent data conflicts and gives users control over their data.

Key Features:
  • Match Confidence: Shows how the duplicate was detected (APN match = exact, Address match = high confidence)
  • Side-by-Side Comparison: Click "Compare" to see uploaded data vs existing data with differences highlighted
  • Per-Row Actions: Each duplicate can be handled independently
  • Bulk Actions: Select multiple rows and apply same action to all
  • Optional Notes: Add context when updating existing records (logged in audit trail)
Industry Alignment:

Follows CDC De-Duplication Best Practices: "front-end matching" where duplicates are identified and presented to users BEFORE insertion. Similar to data import patterns in Salesforce, HubSpot, and CRE platforms like CoStar.
