Purpose & Use Case:

User provisioning and management across all customer accounts. Essential for onboarding new users, managing access, and troubleshooting login issues. This is where you create accounts for your customers' employees.

Key Features Explained:
  • Create User: Adds a new user to an account. Sends welcome email with password setup link.
  • Role Badges: Super Admin (you), Administrator (account admin), User (standard), Data Entry (limited access). Controls permissions.
  • Account Assignment: Each user belongs to one account. Determines what data they can access.
  • Status (Active/Inactive): Inactive users can't login but data is preserved. Use for former employees.
  • Reset Password: Sends password reset email. Common support request.
  • Last Login: Helps identify inactive users who may not need accounts anymore.
  • Filters: By account, role, status. Critical when managing 50+ users across multiple accounts.
Implementation Notes:

MUST BUILD FIRST. Users table with foreign key to accounts. Hash passwords (bcrypt). Email via SendGrid/similar. Roles: Start with just 3 roles (Super Admin, Admin, User) - expand later. Delete button for inactive users should be soft delete. Search can be client-side initially. Edit links to admin-user-edit.html.
