DNS Configuration Instructions

Standard Setup Process:

  1. Customer creates subdomain in their DNS provider (GoDaddy, Namecheap, Cloudflare, etc.)
  2. Add CNAME record:
    Type: CNAME
    Name: portal (or subdomain name)
    Value: account-slug.harken.app
    TTL: 3600 (or default)
  3. Customer enters domain in their account settings
  4. Click "Verify Domain" button
  5. Harken auto-verifies:
    • Checks DNS CNAME record via dig/nslookup
    • Provisions SSL certificate (Let's Encrypt)
    • Enables routing in Cloudflare
    • Status updates to "Active"
  6. Domain goes live - End clients can access via custom domain
‚ö†Ô∏è Common Issues:
  • DNS propagation delay: CNAME records can take 5-60 minutes to propagate. Retry verification if it fails immediately.
  • Wrong CNAME target: Ensure it points to account-slug.harken.app, not just harken.app
  • SSL errors: Let's Encrypt requires DNS to be verified first. If SSL fails, DNS verification likely incomplete.
  • Proxied vs DNS-only: On Cloudflare, ensure DNS record is "DNS only" (gray cloud), not proxied
